# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Данные и типы данных, используемые в приложении

### Товары

Интерфейс описывает структуру товара, получаемого от API. Эти данные отображаются в каталоге на главной странице и в модальном окне карточки товара

```
export interface IProduct {
  id: string;
  title: string;
  description: string;
  image: string;
  category: string;
  price: number;
}
```

### Корзина

Элемент корзины содержит минимальный набор данных, необходимых для отображения в модальном окне корзины и расчета общей стоимости заказа.

```
export interface IBasketItem {
  id: string;
  title: string;
  price: number;
}
```

Состояние корзины включает массив добавленных товаров и их общую сумму. Корзина отображается как модальное окно, доступное с главной страницы.

```
export interface IBasketState {
  items: IBasketItem[];
  total: number;
}
```

### Заказ

Определяет допустимые способы оплаты: банковская карта или наличные при получении.

```
export type TPaymentMethods = 'card' | 'cash';
```

Данные заказа, передаваемые на сервер. Поля формы разбиты на два этапа: адрес и способ оплаты, затем — email и телефон. Валидация обязательна на каждом шаге.

```
export interface IOrder {
  payment: TPaymentMethods;
  address: string;
  email: string;
  phone: string;
  items: string[];
}
```
Ответ сервера на заказ

```
export interface IOrderResult {
  id: string;
  total: number;
}
```
Сервер возвращает идентификатор заказа и итоговую сумму. Эти данные отображаются в финальном окне подтверждения заказа.


## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP: 
- слой представления, отвечает за отображение данных на странице, 
- слой данных, отвечает за хранение и изменение данных
- презентер, отвечает за связь представления и данных.

### Базовый код

В проекте используются базовые классы, расположенные в папке src/components/base/. Они обеспечивают фундаментальную функциональность для работы с API и взаимодействия между компонентами приложения через события.

#### Класс Api

Базовый HTTP-клиент, cодержит в себе базовую логику отправки запросов. В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.
Методы: 
- `get` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер
- `post` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.

#### Класс EventEmitter
Брокер событий позволяет отправлять события и подписываться на события, происходящие в системе. Класс используется в презентере для обработки событий и в слоях приложения для генерации событий.  
Основные методы, реализуемые классом описаны интерфейсом `IEvents`:
- `on` - подписка на событие
- `emit` - инициализация события
- `trigger` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие

### Слой данных

Модели находятся в папке src/components/Model/. Каждая из них отвечает за управление определённым видом данных — товарами, корзиной и заказом.

#### Класс ProductModel

Класс отвечает за работу с каталогом товаров, отображаемых на главной странице.
Конструктор принимает экземпляр брокера событий. EventEmitter, реализующий интерфейс IEvents. Это позволяет модели взаимодействовать с другими компонентами — например, оповещать представление о том, что список товаров обновился, или что пользователь выбрал карточку для предпросмотра.
В полях класса хранятся следующие данные:

- _items: IProduct[] - массив всех товаров, полученных от API
- _preview: string | null - id товара, выбранного для отображения в модальном окне
- events: IEvents - брокер событий

Так же класс предоставляет набор методов для взаимодействия с этими данными.

- setItems(items: IProduct[]): void - устанавливает список товаров и инициирует событие products:changed
- getItems(): IProduct[] - возвращает список товаров
- getItem(id: string): IProduct | undefined - возвращает товар по его id
- setPreview(id: string): void - устанавливает id выбранного товара и инициирует событие preview:changed
- resetPreview(): void - сбрасывает выбор и инициирует preview:reset

#### Класс BasketModel

Отвечает за логику работы с корзиной покупок.
Конструктор принимает экземпляр брокера событий.

В полях класса хранятся следующие данные:

- _items: IBasketItem[] - массив товаров, добавленных в корзину
- events: IEvents - брокер событий

Так же класс предоставляет набор методов для взаимодействия с этими данными.

- addItem(item: IBasketItem): void - добавляет товар в корзину и инициирует событие basket:changed
- removeItem(id: string): void - удаляет товар по id и инициирует basket:changed
- clear(): void - очищает корзину и инициирует basket:changed
- getItems(): IBasketItem[] - возвращает текущие товары
- getTotal(): number - возвращает общую сумму всех товаров

#### Класс OrderModel

Модель отвечает за хранение и подготовку данных заказа, введённых пользователем на двух этапах оформления покупки.
Конструктор принимает экземпляр брокера событий.

В полях класса хранятся следующие данные:

- address: string - адрес доставки, вводится на первом шаге
- payment: TPaymentMethods - способ оплаты ('card' или 'cash')
- email: string - адрес электронной почты, вводится на втором шаге
- phone: string - номер телефона, вводится на втором шаге
- items: string[] - массив идентификаторов товаров из корзины
- events: IEvents - брокер событий

Так же класс предоставляет набор методов для взаимодействия с этими данными:

- setItems(items: string[]): void - сохраняет массив id товаров из корзины
- setStep1(data: { address: string; payment: TPaymentMethods }): void - устанавливает данные первого шага (доставка и способ оплаты), инициирует событие order:step1
- setStep2(data: { email: string; phone: string }): void - устанавливает данные второго шага, инициирует событие order:step2
- getOrder(): IOrder - возвращает финальный объект заказа в формате, ожидаемом API
- reset(): void - сбрасывает все данные заказа, инициирует событие order:reset

### Классы представления
Все компоненты отображения находятся в папке src/components/View/ и отвечают за представление данных в DOM. Они получают данные от моделей и подписываются на события из брокера.

#### Класс ModalView

Базовый компонент модального окна. Используется как универсальный контейнер, внутрь которого динамически вставляется содержимое через шаблоны. Такой подход соответствует принципу композиции.

Конструктор принимает селектор, по которому в разметке страницы будет идентифицировано модальное окно и экземпляр класса `EventEmitter` для возможности инициации событий.

- constructor(modalSelector: string, events: IEvents)

Поля класса:

- modal: HTMLElement - корневой элемент модального окна
- events: IEvents - брокер событий
- content: HTMLElement - контейнер для вставки шаблона

Методы класса:

- setContent(template: HTMLElement): void - очищает текущее содержимое и вставляет переданный DOM-элемент (например, карточку, корзину, форму)
- open(): void - открывает модальное окно, добавляя соответствующие классы и обработчики
- close(): void - закрывает модальное окно и очищает содержимое
- setEventListeners(): void - навешивает слушатели на кнопку закрытия, клавишу Escape и overlay

#### Класс CardView

Отвечает за отображение карточки товара в каталоге на главной странице. Используется для генерации DOM-элемента карточки на основе HTML-шаблона и данных из ProductModel.

Создаёт визуальное представление товара с изображением, названием, категорией и ценой. Также устанавливает обработчики для кликов: открыть модальное окно с подробностями (card:select) и добавить товар в корзину (card:add).

Конструктор принимает параметрами HTML-шаблон карточки товара и экземпляр брокера событий EventEmitter.

Поля класса:

- template: HTMLTemplateElement	- шаблон DOM-структуры карточки
- events: IEvents - брокер событий
- element: HTMLElement - DOM-элемент карточки, сгенерированный из шаблона
- titleElement: HTMLElement - элемент названия товара
- priceElement: HTMLElement - элемент цены
- imageElement: HTMLImageElement - картинка товара
- categoryElement: HTMLElement - элемент с категорией
- addButton: HTMLElement - кнопка "в корзину"
- data: IProduct - хранимые данные товара (для удобства в событиях)

Методы класса:

- setData(product: IProduct): void - заполняет шаблон данными товара

- render(): HTMLElement - возвращает готовый элемент карточки

#### Класс BasketView

Компонент для отображения корзины внутри модального окна. Используется для генерации HTML-содержимого модального окна с корзиной товаров. Отвечает за отображение списка добавленных товаров, их названий и цен, а также итоговой суммы заказа. Предоставляет разметку, которую вставляет универсальный контейнер ModalView с помощью метода setContent().

Конструктор принимает параметрами HTML-шаблон карточки товара и экземпляр брокера событий EventEmitter.

Поля класса:

- template: HTMLTemplateElement - сылка на HTML-шаблон корзины
- events: IEvents - брокер событий, через который передаются действия пользователя
- element: HTMLElement - корневой DOM-элемент (сгенерированный из шаблона)
- listElement: HTMLElement - элемент внутри шаблона, в который вставляются товары
- totalElement: HTMLElement - элемент, отображающий итоговую сумму
- buttonElement: HTMLButtonElement - кнопка "Оформить"

Методы класса: 

- render() - очищает предыдущий список товаров, генерирует DOM-элементы для каждого товара, выводит общую сумму заказа, возвращает готовый HTML для вставки в модальное окно.
```
render(items: IBasketItem[], total: number): HTMLElement.
```

#### Класс FormStep1View

Компонент формы первого шага оформления заказа: ввод адреса доставки и выбор способа оплаты. Отображает форму с полем "Адрес доставки" и двумя переключателями оплаты: картой или наличными. Валидирует ввод и управляет доступностью кнопки "Далее".

Конструктор принимает параметрами HTML-шаблон карточки товара и экземпляр брокера событий EventEmitter.

Поля класса:

- template: HTMLTemplateElement - шаблон формы
- events: IEvents - брокер событий
- element: HTMLFormElement - корневая форма
- addressInput: HTMLInputElement - поле ввода адреса
- paymentInputs: NodeListOf<HTMLInputElement> - переключатели оплаты
- submitButton: HTMLButtonElement - кнопка "Далее"

Методы класса:

- getInputValues(): { address: string; payment: TPaymentMethods } - возвращает объект с адресом и способом оплаты.

- setValid(isValid: boolean): void - активирует или деактивирует кнопку "Далее".

Также добавляются слушатели на ввод/изменение, генерирующие событие order:step1:input, и на отправку формы - событие order:step1:submit.

#### Класс FormStep2View

Конструктор принимает параметрами HTML-шаблон карточки товара и экземпляр брокера событий EventEmitter.

Поля класса:

- events: IEvents - брокер событий

Методы класса:

#### Класс ModalSuccessView

Простой компонент, отображающий сообщение об успешном оформлении заказа. Выводит сообщение "Заказ успешно оформлен" и кнопку "Закрыть". Используется как финальный экран оформления заказа.

Конструктор принимает параметрами HTML-шаблон карточки товара и экземпляр брокера событий EventEmitter.

## Взаимодействие компонентов

Код, описывающий взаимодействие представления и данных между собой находится в файле `index.ts`, выполняющем роль презентера.\
Взаимодействие осуществляется за счет событий генерируемых с помощью брокера событий и обработчиков этих событий, описанных в `index.ts`\
В `index.ts` сначала создаются экземпляры всех необходимых классов, а затем настраивается обработка событий.

### События моделей (генерируются из слоя данных)

- `products:changed` -	cписок товаров обновлён
- `preview:changed`	- открыт просмотр карточки
- `preview:reset` - закрыто модальное окно карточки
- `basket:changed` - список товаров в корзине или сумма изменились
- `order:step1`	- установлены данные доставки и оплаты
- `order:step2`	- установлены контакты
- `order:reset`	- заказ сброшен

### События пользовательского интерфейса (из слоя View)

- `card:select`	- пользователь кликнул на карточку товара
- `card:add` - товар добавлен в корзину
- `card:preview:close` - закрыть модалку карточки
- `basket:open`	- открыта корзина
- `basket:item:remove` - удалён товар из корзины
- `basket:checkout`	- нажата кнопка “Оформить заказ”
- `order:step1:input` - изменены данные формы доставки
- `order:step1:submit` - отправлена форма доставки
- `order:step2:input` - изменены данные формы контактов
- `order:submit` - подтверждён заказ (все данные валидны)
- `order:done` - заказ успешно оформлен и подтверждение закрыто

### Примеры связи

- Когда пользователь выбирает товар (card:select), в модель передаётся setPreview(id) → срабатывает preview:changed → в модалку подставляется карточка товара.

- При клике "добавить в корзину" → card:add → BasketModel.addItem() → срабатывает basket:changed → иконка и модалка обновляют данные.

- После заполнения форм шага 1 и 2 → order:step1:submit и order:step2:submit → данные сохраняются в OrderModel → при order:submit вызывается api.postOrder() → отображается окно успеха.
